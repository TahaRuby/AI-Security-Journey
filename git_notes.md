# gitnote — یادداشت دستورهای گیت

> این فایل رو هر بار که یه دستور گیت جدید یاد گرفتی یا استفاده کردی، تکمیل کن.
> برای هر دستور: توضیح رو بخون (یا خودت بازنویسی کن)، و قسمت "مثال من" رو با تجربه واقعی خودت پر کن.

---

## بخش ۱ — کارهای روزمره (Basic)

### git init
**توضیح:** یک ریپوی گیت جدید توی پوشه فعلی می‌سازه.
**مثال من:** _______________
### git status
**توضیح:** نشون می‌ده چه فایل‌هایی تغییر کردن، کدوم‌ها آماده کامیت هستن و کدوم‌ها نه.
**مثال من:** _______________

### git add
**توضیح:** فایل(های) تغییر یافته رو برای کامیت بعدی آماده می‌کنه (staging).
**مثال من:** _______________

### git commit -m "پیام"
**توضیح:** تغییرات آماده‌شده رو به‌صورت یک نسخه (snapshot) با یک پیام توضیحی ذخیره می‌کنه.
**مثال من:** _______________

### git push
**توضیح:** کامیت‌های محلی رو به ریپوی روی گیت‌هاب (یا هر سرور دیگه) ارسال می‌کنه.
**مثال من:** _______________

### git pull
**توضیح:** آخرین تغییرات ریپوی روی سرور رو می‌گیره و با نسخه محلی ادغام می‌کنه.
**مثال من:** _______________

### git clone <آدرس>
**توضیح:** یک کپی کامل از یک ریپوی موجود (مثلاً از گیت‌هاب) رو روی سیستم خودت می‌سازه.
**مثال من:** _______________

### git log
**توضیح:** تاریخچه کامیت‌ها رو نشون می‌ده (کی، چه زمانی، چه پیامی).
**مثال من:** _______________

### git branch
**توضیح:** لیست شاخه‌های موجود رو نشون می‌ده، یا با اسم بعدش یک شاخه جدید می‌سازه.
**مثال من:** _______________

### git checkout <نام‌شاخه>
**توضیح:** بین شاخه‌های موجود جابه‌جا می‌شه.
**مثال من:** _______________

### git checkout -b <نام‌شاخه>
**توضیح:** یک شاخه جدید می‌سازه و بلافاصله بهش سوییچ می‌کنه.
**مثال من:** _______________

### git merge <نام‌شاخه>
**توضیح:** تغییرات یک شاخه دیگه رو با شاخه فعلی ادغام می‌کنه.
**مثال من:** _______________

**توضیح:** تغییرات کامیت‌نشده رو موقتاً کنار می‌ذاره تا بتونی بدون از دست دادنشون به کار دیگه‌ای بری.
**مثال من:** _______________

### git stash pop
**توضیح:** آخرین تغییراتی که stash کرده بودی رو برمی‌گردونه.
**مثال من:** _______________

### git reset <فایل>
**توضیح:** یک فایل رو از حالت staged خارج می‌کنه (بدون پاک کردن تغییرات).
**مثال من:** _______________

### git revert <کامیت>
**توضیح:** یک کامیت قبلی رو با ساختن یک کامیت جدید خنثی می‌کنه (بدون پاک کردن تاریخچه).
**مثال من:** _______________
